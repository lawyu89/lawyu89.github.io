Introduction to Version Control, Git, and GitHub

Lessons of my first week in phase 0

1/16/2015

First week of phase 0 in DBC has been interesting to say the least. I've already went through a ton of computer topics and commands which I'm happy to reiterate some of it here to help out anyone that might be interested in learning or thinking about switching to computer science.

First, I'll go over is something called version control. Version control system is something developers use to frequently make save points of a code to record all the changes made to it. That way if you made a error in your code somewhere along the way, you can go back to an earlier save point and figure out just what went wrong. Many develops uses version control tools of some sort, Git being one of the more popular ones.

Git is an open-source version control tool you can use in your computer (open-source means that the source/code is available for everyone to see, edit, or use however they see fit). Git allows the user to view code files in like in branches. The master branch stores your main code but you can have off branches that takes a copy of the whole or parts of master branch. With these off branches you can make changes to the code (called commits). These commits tracks all the changes you made. After you thoroughly reviewed changes made, you can apply the commits made and then merge it with your master branch. Git is so widely used, there's even a way to transfer your codes from your local computer to a remote server using GitHub.

GitHub is a platform that developers to can use to remotely store their code and save their progress. It's similar to git except the files are stored remotely on a server rather than inside your computer. Since the files are stored on a server, you can edit and commits changes from your computer and then send it to GitHub. The major benifit of GitHub is that is allows multiple people to access the same code and work on it collaboratively without being on the same computer! This is incredibly useful for large projects since you can have different people working on different parts of a project working from different parts of the world instead of everyone being in the same room.

Well that it for this week on things I learned this from week 1 of phase 0 at DBC. Stay tuned for next weeks topics as we cover more advance material!
