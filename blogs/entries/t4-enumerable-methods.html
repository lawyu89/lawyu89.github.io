Ruby Enumerable:group_by

What it does and how to use it!

2/2/15

Hello there! This week’s technical blog will be about the group_by enumerable. The group_by enumerable is very useful tool you can use to sort information and store that information into a dictionary-like collection hash. So how does the group_by enumerable actually work and how do we use it? Like most other enumerables, you call on it by placing a period after your array and type the group_by command along with a block to tell it how you want to separate the list by ImAnArray.group_by {|obj| block}.

Now that I’ve gone through how to call on the method, let’s go over what happens when you actually use it. First you need to declare your object name closed in the pipe lines. The object name is simply a variable that references the current element in the array it is looking since the group_by method iterates over each element. After declaring the object name, you need to enter a code block that tells the method how you want to separate the array. For example lets say we have the following array: array=[0,1,1,2,4,7]. If we wanted to split the array into a hash and group them by even and odds we can do something like array.group_by {|num| num%2==0} which will yield:{true=>[0,2,4], false=>[1,1,7]. Our keys will be "true" and "false" (the keys may value based on the condition, in this example the condition was a boolean experssion), each associated with a value thats an array of [0,2,4] and [1,1,7] respectively

As you can see, group_by is a power tool you can use to sort arrays into a key-value pair hash. Now that you have a better idea of the group_by enumerable, go on and try it out yourself!
